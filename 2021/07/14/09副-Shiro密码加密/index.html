前言

狂神的课程和笔记中只能说是对Shiro的基本使用有所介绍，但实际上工作中不可能只有这么些简单的使用，因此笔者通过互联网上的资源进行了进一步学习，本文将记录学习中进行配置和一些测试的过程。

本文中使用MD5加salt进行密码加密，得到的为32。MD5和salt的使用算是够用。本人也对Sha512+salt进行了测试，重复过程不进行记录。读者也可以尝试使用其他的加密方式进行加密。

本文的测试用例项目github库地址：shiroDemo

开始配置吧！

下面开始来进行配置步骤！

获取一个加密的密码

本文的测试用例沿用09-集成shiro中所使用的项目，不额外添加一个添加（注册）新用户的模块，所以先直接使用一个SpringBootTest的测试用例获取一个加密好的密文，将数据库中的示例用户密码直接修改。

  • 获取密文的测试类

    一般而言，盐不应该采用固定的字符串，而是使用username作为盐。我觉得使用username也不安全，使用用户的UUID或者其他一些用户相关的复杂点的信息作为盐更加安全些吧。本文中由于是测试需求，就不设置太过麻烦的加盐规则了。

    @SpringBootTest
class ShiroDemoApplicationTests {
    @Test
    void getMD5Pwd() {
        //对密码"admin"和盐"salt01"进行MD5加密10次
        Md5Hash pwd = new Md5Hash("admin","salt01",10);
        
        //打印输出得到的密文
        System.out.println("pwd = " + pwd +"  长度"+ pwd.toString().length());

    }
}

    得到结果如下：

  • 将数据库中的表的password进行一些修改，改为char类型长度32位，以提高进行检索时的效率

  • 将得到的密文复制到数据库的password属性中

UserRealm.java 中的配置

@Override
protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken token) throws AuthenticationException {
    System.out.println("执行了=>认证逻辑AuthenticationToken");
    
    UsernamePasswordToken userToken = (UsernamePasswordToken) token;
    //连接数据库
    //查询
    User user = userService.getUserByName(userToken.getUsername());
    //查询结果为空时，返回null，控制器会抛出无用户的异常
    if (user == null) {
        return null;
    }
    //配置加密盐的ByteSource
    ByteSource credentialsSalt = ByteSource.Util.bytes("salt01");
    /*返回属性依次为：
		user: 用户的全部信息
		user.getPassword(): 用户的正确密码
		credentialsSalt: 加密的盐
		"UserRealm": realmName
    */
    return new SimpleAuthenticationInfo(user, user.getPassword(),credentialsSalt,"UserRealm");
}

ShiroConfig.java中的配置

//创建 realm 对象，需要自定义类
@Bean
public UserRealm userRealm() {
    UserRealm userRealm = new UserRealm();
    //设置加密方式
    HashedCredentialsMatcher hashedCredentialsMatcher = new HashedCredentialsMatcher();
    //设置加密工具
    hashedCredentialsMatcher.setHashAlgorithmName("MD5");
    //设置加密次数
    hashedCredentialsMatcher.setHashIterations(10);
    //将加密配置添加到UserRealm的设置中
    userRealm.setCredentialsMatcher(hashedCredentialsMatcher);
    return userRealm;
}

其实这一步代码中的6~12行也可以在UserRealm.java或者ShiroConfig.java的getDefaultWebSecurityManager()中进行配置。

理解了这些东西的加载步骤就能更灵活的去进行配置了。
